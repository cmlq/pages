## 1、**基础认知**

### **(1)、场景：用户登录一个网站**

1. 你在前端页面（看得见的界面）上，输入账号密码，点击【登录】。

2. 前端把账号密码打包成一个 “请求”（Request），通过 API（约定好的地址和格式）发送给后端。

3. 后端收到请求，去 数据库（存数据的地方）里查：有没有这个账号？密码对不对？

4. 数据库把查询结果（“有”或“没有”）返回给后端。

5. 后端根据结果做出判断：

    - 如果正确：生成一个“登录成功”的 “响应”（Response）。

    - 如果错误：生成一个“密码错误”的响应。

6. 后端通过 API 把这个响应返回给前端。

7. 前端收到响应，在界面上告诉你：“登录成功，跳转首页” 或 “密码错误，请重试”。

流程：

前端发问 → API传话 → 后端处理 → 数据库存取 → 原路返回 → 前端显示

# **第二部分：前端基础技术（基础入门）**

前端技术核心是“构建用户可交互的界面”，底层依赖HTML、CSS、JavaScript三大基础技术，其中div、布局、组件、CSS均属于前端基础核心内容，从底层结构到样式呈现，逐步拆解讲解。



## **一、前端基础组件介绍**

### **组件底层定义**

前端组件是“HTML结构\+CSS样式\+JavaScript交互”的组合体，底层逻辑是“封装可复用的界面单元”，将重复出现的界面模块（如tab切换、按钮、弹窗）封装成组件，可多次复用，减少重复开发，提升开发效率。

#### HTML 是骨架

- 负责**页面有什么**：标题、按钮、文字区域

- `div` = 盒子，用来装内容

- `id` 给元素起名字，方便 JS/CSS 找到它

#### CSS 是外观

- 负责**长什么样**：大小、颜色、边框、居中、隐藏

- `#title` → 找 id 为 title 的元素

- `.box` → 找 class 为 box 的元素

- `display: none` → 隐藏元素

#### JS 是行为

- 负责**做什么**：点击、变色、显示隐藏

- `document.getElementById()` → 找到元素

- `onclick` → 点击触发函数

- `if/else` → 判断逻辑

## **二、前端底层基础：HTML核心元素（div及基础结构）**

### **1. HTML底层定位**

HTML（超文本标记语言）是前端界面的“骨架”，负责定义界面的结构和内容，所有前端可视元素都依赖HTML标签搭建，无需编译，直接由浏览器解析渲染。底层逻辑：通过标签（标记）告知浏览器“某块区域是什么内容、是什么类型”，是前端技术的基础载体。

```HTML
<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <title>前端基础教学案例</title>
</head>
<body>
<!--主体内容-->
<div class="box">
    <h2 id="title">我是标题</h2>
    <button onclick="changeColor()">点击变色</button>
    <button onclick="toggleShow()">显示/隐藏内容</button>
    <div id="content">我是被显示隐藏的内容！</div>
</div>
</body>
</html>
```


### **2. div标签（前端布局核心元素）**

div是HTML中最基础、最常用的“块级元素”，底层作用是“划分界面区域、承载其他元素”，是前端布局的核心基石，无默认样式（默认占满一行，无边框、无内边距），需配合CSS设置样式和布局。

核心特性：

- 块级元素：默认独占一行，宽度默认等于父容器宽度，高度由内部内容决定；

- 可嵌套性：可嵌套其他所有HTML元素（包括div、文本、图片、按钮等），实现复杂界面的区域划分；

- 无默认样式：本身不具备任何视觉效果，需通过CSS设置宽高、边框、背景、边距等样式，才能呈现可视效果；

- 可通过id、class属性标识，方便CSS定位样式、JavaScript操作元素。

底层应用场景：所有前端界面的区域划分（如顶部导航区、左侧菜单区、中间内容区、底部版权区），均通过div搭建基础结构，再配合其他标签完善内容。

## **三、前端样式基础：CSS**

### **1. CSS底层定位**

CSS（层叠样式表）是前端界面的“皮肤”，底层作用是“美化HTML结构”，控制元素的视觉效果（颜色、大小、边框、间距、字体等），与HTML分离，实现“结构与样式分离”，便于维护和修改。

底层逻辑：浏览器解析HTML后，再解析CSS，将CSS样式应用到对应的HTML元素上，呈现最终的可视界面，CSS的“层叠性”（优先级）决定了多个样式冲突时，哪个样式会生效。



### **2. CSS核心基础**

- 选择器：用于定位需要设置样式的HTML元素，是CSS作用于元素的“桥梁”，核心常用选择器：id选择器（#id）、class选择器（.class）、元素选择器（div、p等）、后代选择器（div p）；

- 核心样式属性：宽高（width、height）、边框（border）、边距（margin、padding）、背景（background）、字体（font-size、font-family）、颜色（color）、定位（position、float）、弹性布局（display: flex）；

- 层叠性：当多个CSS样式作用于同一个元素时，按“优先级”生效（id选择器 > class选择器 > 元素选择器），相同优先级时，后定义的样式覆盖先定义的样式；

- 继承性：子元素会继承父元素的部分CSS样式（如字体、颜色），减少重复代码，底层目的是提高开发效率。

### **案例：tab组件（基础交互组件）底层实现**

tab组件核心功能是“切换显示不同内容”，底层由3部分组成，无复杂逻辑，是入门级核心组件：

- HTML结构：用div搭建tab导航区（多个tab选项）和tab内容区（多个内容面板），每个tab选项对应一个内容面板，通过id或class关联；

- CSS样式：设置tab导航的默认样式、选中样式（如选中时字体加粗、添加下划线），隐藏未选中的内容面板（display: none）；

- JavaScript交互：给每个tab选项绑定点击事件，点击时，切换tab选中样式，显示对应的内容面板，隐藏其他内容面板，实现“点击切换”的核心功能。

补充：其他基础组件（按钮、输入框、弹窗）的底层逻辑与tab组件一致，均是“结构+样式+简单交互”的组合，核心是“复用性”。

<head>

        }

        #content {
            margin-top: 15px;
            display: none; /* 默认隐藏 */
            color: green;
        }
    </style>
</head>
<body>

    <!-- HTML 结构：页面骨架 -->
    <div class="box">
        <h2 id="title">我是标题</h2>
        <button onclick="changeColor()">点击变色</button>
        <button onclick="toggleShow()">显示/隐藏内容</button>
        <div id="content">我是被显示隐藏的内容！</div>
    </div>

    <script>
        // JS 部分：实现交互功能
        // 1. 切换标题颜色
        function changeColor() {
            let title = document.getElementById("title");
            // 如果是蓝色 → 改成红色，否则 → 改成蓝色
            if (title.style.color === "red") {
                title.style.color = "blue";
            } else {
                title.style.color = "red";
            }
        }

        // 2. 显示/隐藏内容
        function toggleShow() {
            let content = document.getElementById("content");
            // 如果隐藏 → 显示；如果显示 → 隐藏
            if (content.style.display === "none") {
                content.style.display = "block";
            } else {
                content.style.display = "none";
            }
        }
    </script>

</body>
</html>

五、F12 调试
1. 如何打开开发者工具
- Windows：直接按 F12 或 Ctrl + Shift + I
- Mac：Cmd + Option + I
- 备选：右键点击网页任意位置 → “检查” 或 “审查元素”
💡 打开后通常出现在页面右侧或底部，可以点击“三个点”菜单调整停靠位置。

---
2. 认识四个最常用的面板（只讲这些）
- Elements（元素）：查看网页的 HTML 结构，临时修改文字/图片/样式。
- Console（控制台）：看红色报错信息，复制报错给开发人员。
- Network（网络）：检查接口是否成功、加载慢在哪里。
- 手机图标（设备模拟）：临时看手机版页面效果。
⚠️ 重要：所有修改都是临时的，刷新页面就恢复，不会影响线上。

---
3. Elements 面板：改文案、查布局
- 改文字：点击箭头图标 → 点选页面上的文字 → 在右侧 HTML 中双击文字，直接输入新内容 → 回车（截图用）。
- 隐藏元素：选中元素后按 Delete 或 H 键隐藏。
- 修改颜色/字体：右侧 Styles 栏里点击颜色或数值直接改。
🎯 典型场景：运营想预览 banner 文案换几个字的效果，无需找开发。

---
4. Console 面板：看报错、复制错误
- 红色文字就是报错，把内容截图或复制给开发。
- 清空：左上角“禁止”图标（清空控制台）。
- 简单命令（可选）：输入 document.title 回车可以看页面标题。
🎯 典型场景：点按钮没反应 → 打开 Console，看到红色报错 → 截图反馈。

---
5. Network 面板：检查接口和图片
- 录制开关：默认红色圆点表示在录制，刷新页面后所有请求都会列出。
- 看某个请求是否成功：点击请求 → 看 Status 列，200 为成功，404/500 为失败。
- 看接口返回数据：点击请求 → 右侧选择 Preview 或 Response 选项卡。
- 清空请求：左上角“禁止”图标。
🎯 典型场景：提交表单没反应 → 打开 Network，刷新或重新提交，找到那个请求，看状态码和返回结果。

---
6. 手机模拟器：快速看移动端效果
- 打开方式：点击 F12 左上角的“手机+平板”图标（或按 Ctrl+Shift+M）。
- 切换机型：顶部选择 iPhone 或 Android 型号。
- 刷新：模拟真实手机加载。
🎯 典型场景：产品经理想知道手机端某个按钮是否遮挡内容，不用真机。

---
7. 常用小技巧
- 强制刷新（清除缓存）：Ctrl + F5 或 Cmd + Shift + R（避免样式/图片不更新）。
- 搜索：在 Elements 或 Network 中按 Ctrl + F 搜索文字。
- 关闭工具：按 F12 或点击右上角 ✕。

---
8. 注意事项（避免犯错）
- 不要乱改：改完 Elements 刷新即丢，别担心。
- 不要认为这就是最终效果：自己改的文字/样式只是本地预览，线上没变。
- 报错不一定都是前端问题：Network 看到 500 错误是后端服务问题，截图给开发即可。

### 2、HTML格式规范

```HTML
<!DOCTYPE html> <!--文档类型声明，告诉浏览器：这个页面使用的是 HTML5 标准-->
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">

</head>
<body>
<!--内容区域-->

<!--JS区域-->
</script>
</body>
</html>
```

#### 2.1 HTML 规范（页面结构）

##### 2.1.1 标签规范

- 标签必须闭合

    - 正确：`<div></div>` `<input />`

    - 错误：`<div>` 不闭合

- 标签嵌套规范：块级包行内，不乱嵌套

- 不使用废弃标签（如 `<center>`）

##### 2.1.2 属性规范

- `id` 唯一：同一个页面不能重复

- `class` 可重复，用于样式

- 属性值必须加双引号

    - 正确：`<div id="loginBox">`

    - 错误：`<div id=loginBox>`

- 自定义属性用 `data-` 开头：`data-id="1001"`

#### 2.2 CSS 规范（样式）

##### 2.2.1 书写规范

- 每条样式占一行

- 选择器命名见名知意：`.login-form` `.btn-submit`

- 颜色用十六进制：`#fff` `#333`

##### 2.2.2 避免问题

- 不写冗余样式：重复设置无用属性

- 不使用行内样式优先：`<div style="color:red">`（尽量少用）

- 样式统一管理，不乱写

### 3、通用规范

#### 3.1 命名规范（最重要！90% 的错误来自命名）

- 必须用英文，禁止中文、禁止空格、禁止特殊符号

- 禁止：`姓名`、`用户 name`、`提交！`

- 推荐：`username`、`submitForm`、`userList`

- 统一规则： 

    - 小驼峰：`userName`（JS、ID 常用）

    - 短横线：`user-list`（class 常用）

- 见名知意：看到名字就知道用途 

    - 好：`loginBtn`（登录按钮）

    - 坏：`btn1`、`a`、`div2`


[第二十一页]
#### 3.2 格式规范

- 代码缩进统一（用 2 个空格或 4 个空格）

- 一对标签占一行，不要挤在一起

- 注释清晰：`// 这里是表单提交校验`

- 禁止一行写超长代码，AI 生成的也要换行

#### 3.3 结构规范

- HTML 只负责结构（页面骨架）

- CSS 只负责样式（颜色、大小、布局）

- JS 只负责交互（点击、校验、请求）

- 三者不混写，结构清晰

### 4、禁止

- 中文命名：`<div id="姓名">`

- id 重复：两个元素 `id="user"`

- 无意义命名：`a` `b` `btn1` `div1`

- 标签不闭合：`<div></div>`

- 函数不注释，看不懂用途

- 样式混乱、行内样式满天飞

- JS 变量重复定义

- 代码挤在一行，无法阅读
