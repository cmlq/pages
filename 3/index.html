基础认知

基础认知

通过“用户登录一个网站”的场景，理解前端、API、后端和数据库如何配合工作。

03/ 21
场景

用户登录一个网站

把一次登录操作拆成前端、API、后端和数据库四个环节来理解。

1
在前端页面输入账号密码，点击【登录】。
2
前端把账号密码打包成一个请求，通过 API 发送给后端。
3
后端收到请求后，去数据库里查账号和密码是否正确。
4
数据库把查询结果返回给后端，后端开始判断结果。
流程

整条链路怎么走

前端发问 → API传话 → 后端处理 → 数据库存取 → 原路返回 → 前端显示

5
如果账号密码正确，就返回登录成功。
6
如果账号密码错误，就返回密码错误。
7
前端收到结果后，提示登录成功或请重试。
四个角色
  • 前端：展示界面，接收输入。
  • API：约定好的通道，负责传递请求和响应。
  • 后端：处理业务逻辑，做判断。
  • 数据库：存放和查询数据。
学习顺序

先看清每个角色做什么，再把它们连成一条完整的登录流程。

记住一句话

前端负责展示和交互，API 负责传递，后端负责处理，数据库负责存取。

把这个链路记住，后面学习接口、请求和响应会更容易理解。

前端基础技术（基础入门）

05/ 21

前端技术核心是“构建用户可交互的界面”，底层依赖HTML、CSS、JavaScript三大基础技术，其中div、布局、组件、CSS均属于前端基础核心内容，从底层结构到样式呈现，逐步拆解讲解。

一、前端基础组件介绍

组件底层定义

前端组件是“HTML结构+CSS样式+JavaScript交互”的组合体，底层逻辑是“封装可复用的界面单元”，将重复出现的界面模块（如tab切换、按钮、弹窗）封装成组件，可多次复用，减少重复开发，提升开发效率。

HTML 是骨架

  • 负责页面有什么：标题、按钮、文字区域
  • div = 盒子，用来装内容
  • id 给元素起名字，方便 JS/CSS 找到它

CSS 是外观

  • 负责长什么样：大小、颜色、边框、居中、隐藏
  • #title → 找 id 为 title 的元素
  • .box → 找 class 为 box 的元素
  • display: none → 隐藏元素

JS 是行为

  • 负责做什么：点击、变色、显示隐藏
  • document.getElementById() → 找到元素
  • onclick → 点击触发函数
  • if/else → 判断逻辑
第二部分

HTML核心元素（div及基础结构）

06/ 21

HTML（超文本标记语言）是前端界面的“骨架”，负责定义界面的结构和内容，所有前端可视元素都依赖HTML标签搭建，无需编译，直接由浏览器解析渲染。底层逻辑：通过标签（标记）告知浏览器“某块区域是什么内容、是什么类型”，是前端技术的基础载体。

HTML 示例代码

HTML
<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <title>前端基础教学案例</title>
</head>
<body>
<!--主体内容-->
<div class="box">
    <h2 id="title">我是标题</h2>
    <button onclick="changeColor()">点击变色</button>
    <button onclick="toggleShow()">显示/隐藏内容</button>
    <div id="content">我是被显示隐藏的内容！</div>
</div>
</body>
</html>

div标签（前端布局核心元素）

div是HTML中最基础、最常用的“块级元素”，底层作用是“划分界面区域、承载其他元素”，是前端布局的核心基石，无默认样式（默认占满一行，无边框、无内边距），需配合CSS设置样式和布局。

核心特性

  • 块级元素：默认独占一行，宽度默认等于父容器宽度，高度由内部内容决定；
  • 可嵌套性：可嵌套其他所有HTML元素（包括div、文本、图片、按钮等），实现复杂界面的区域划分；
  • 无默认样式：本身不具备任何视觉效果，需通过CSS设置宽高、边框、背景、边距等样式，才能呈现可视效果；
  • 可通过id、class属性标识，方便CSS定位样式、JavaScript操作元素。

底层应用场景

所有前端界面的区域划分（如顶部导航区、左侧菜单区、中间内容区、底部版权区），均通过div搭建基础结构，再配合其他标签完善内容。

案例：tab组件（基础交互组件）

08/ 21

tab 组件核心功能

tab组件核心功能是“切换显示不同内容”，底层由3部分组成，无复杂逻辑，是入门级核心组件。

HTML结构
用div搭建tab导航区（多个tab选项）和tab内容区（多个内容面板），每个tab选项对应一个内容面板，通过id或class关联。
CSS样式
设置tab导航的默认样式、选中样式（如选中时字体加粗、添加下划线），隐藏未选中的内容面板（display: none）。
JavaScript交互
给每个tab选项绑定点击事件，点击时，切换tab选中样式，显示对应的内容面板，隐藏其他内容面板，实现“点击切换”的核心功能。

补充说明

其他基础组件（按钮、输入框、弹窗）的底层逻辑与tab组件一致，均是“结构+样式+简单交互”的组合，核心是“复用性”。

1. CSS底层定位

CSS（层叠样式表）是前端界面的“皮肤”，底层作用是“美化HTML结构”，控制元素的视觉效果（颜色、大小、边框、间距、字体等），与HTML分离，实现“结构与样式分离”，便于维护和修改。

底层逻辑：浏览器解析HTML后，再解析CSS，将CSS样式应用到对应的HTML元素上，呈现最终的可视界面，CSS的“层叠性”（优先级）决定了多个样式冲突时，哪个样式会生效。

2. CSS核心基础

  • 选择器：用于定位需要设置样式的HTML元素，是CSS作用于元素的“桥梁”，核心常用选择器：id选择器（#id）、class选择器（.class）、元素选择器（div、p等）、后代选择器（div p）；
  • 核心样式属性：宽高（width、height）、边框（border）、边距（margin、padding）、背景（background）、字体（font-size、font-family）、颜色（color）、定位（position、float）、弹性布局（display: flex）；
  • 层叠性：当多个CSS样式作用于同一个元素时，按“优先级”生效（id选择器 > class选择器 > 元素选择器），相同优先级时，后定义的样式覆盖先定义的样式；
  • 继承性：子元素会继承父元素的部分CSS样式（如字体、颜色），减少重复代码，底层目的是提高开发效率。

JS基础知识

11/ 21

1. 基础认知

JS 是什么？
JavaScript（简称 JS）是运行在浏览器里的脚本语言，作用是让静态页面动起来，实现交互、判断、计算、数据处理。
JS 在哪里写？
可以直接写在网页 `<script>` 标签里，也可以写在单独的 `.js` 文件中。
JS 能做什么？
点击按钮弹出提示、获取用户输入内容、判断内容是否为空或格式是否正确、显示 / 隐藏内容、发送数据给后端、页面动态渲染数据。

一句话记忆

JS 是前端页面的“动作控制器”。

    <!-- HTML 结构：页面骨架 -->
    <div class="box">
        <h2 id="title">我是标题</h2>
        <button onclick="changeColor()">点击变色</button>
        <button onclick="toggleShow()">显示/隐藏内容</button>
        <div id="content">我是被显示隐藏的内容！</div>
    </div>

    <script>
        // JS 部分：实现交互功能
        // 1. 切换标题颜色
        function changeColor() {
            let title = document.getElementById("title");
            // 如果是蓝色 → 改成红色，否则 → 改成蓝色
            if (title.style.color === "red") {
                title.style.color = "blue";
            } else {
                title.style.color = "red";
            }
        }

        // 2. 显示/隐藏内容
        function toggleShow() {
            let content = document.getElementById("content");
            // 如果隐藏 → 显示；如果显示 → 隐藏
            if (content.style.display === "none") {
                content.style.display = "block";
            } else {
                content.style.display = "none";
            }
        }
    </script>

</body>
</html>

五、F12 调试

17/ 21

1. 如何打开开发者工具

  • Windows：按 `F12` 或 `Ctrl + Shift + I`。
  • Mac：按 `Cmd + Option + I`。
  • 打开后可以查看网页结构、样式、报错和请求。

2. 常见面板说明

  • `Elements`：看结构和样式。
  • `Console`：看日志、报错、调试 JS。
  • `Network`：看请求、状态码、接口返回。
  • `Sources`：看源码、打断点调试。

3. 调试方法

  • 页面效果不对，先看 `Elements` 里的结构和样式。
  • 按钮没反应，先看 `Console` 有没有报错。
  • 页面请求失败，去 `Network` 看状态码和返回内容。

4. 调试建议

不要认为这就是最终效果，自己改的文字和样式只是本地预览，线上没变。遇到 `500` 报错时，通常是后端服务问题，截图给开发即可。

多用 F12，不要害怕它，前端排查问题很大一部分都要靠它。

通用规范

21/ 21

1 命名规范

  • 必须用英文，禁止中文、空格和特殊符号。
  • 推荐使用 `username`、`submitForm`、`userList`。
  • 小驼峰：`userName`，短横线：`user-list`。
  • 见名知意，看到名字就知道用途。

2 格式规范

  • 代码缩进统一，使用 2 或 4 个空格。
  • 一对标签占一行，不要挤在一起。
  • 注释清晰，函数和变量不注释会看不懂。

3 结构规范

  • HTML 只负责结构。
  • CSS 只负责样式。
  • JS 只负责交互。
  • 三者不混写，结构清晰。

4. 禁止项

  • 中文命名、重复 `id`、无意义命名。
  • 标签不闭合、样式混乱、行内样式满天飞。
  • JS 变量重复定义、代码挤成一行、无法阅读。
