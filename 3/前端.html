01 / 基础认知

用户登录网站的完整流程

用一个最常见的登录场景，把前端、后端、数据库和 API 的关系串起来。

前端负责接收输入和展示结果，后端负责处理逻辑和判断，数据库负责保存和查询数据，API 负责把请求和响应在各层之间传递。

  • 1前端页面输入账号密码，点击登录。
  • 2前端把数据包装成请求，通过 API 发送给后端。
  • 3后端去数据库里查账号和密码是否正确。
  • 4数据库返回结果，后端据此生成响应，再返回给前端。
流程
前端发问 → API传话 → 后端处理 → 数据库存取 → 原路返回 → 前端显示
这是理解 Web 应用最基础的一条链路，也是后续学习接口、状态和数据流的入口。
问答提示
改邮箱为什么不是只改前端？
因为“改完后下次打开还是新的”意味着数据要落到后端和数据库，不只是页面上的临时显示。
前端：输入框 + 点击按钮 API：传递请求 后端：接收并处理 数据库：保存和查询 前端：展示结果
02 / 前端基础

HTML：负责页面内容结构

先讲结构层，先搞清楚页面“有什么”。

HTML 通过标签和语义定义页面骨架，决定标题、段落、按钮、输入框、图片等内容如何组织。

  • 1标签对：有开始和结束，比如 ``。
  • 2固定结构：`` 是页面的基础骨架。
  • 3语义化：标题、段落、按钮、输入框各用各的标签。
  • 4HTML 只负责内容结构，不负责样式和交互。

03 / 前端基础

CSS：负责页面样式和布局

第二页只讲样式层，重点理解页面“长什么样”。

CSS 负责颜色、间距、大小、布局和响应式，是把 HTML 的内容变成可阅读、可展示界面的关键。

  • 1控制外观：颜色、边距、字体、背景。
  • 2控制布局：盒模型、对齐、排列、响应式。
  • 3可以把同一份 HTML 变成不同风格。
  • 4简写属性可以减少重复写法。

04 / 前端基础

JavaScript：负责交互、状态和请求

第三页单独讲 JS，不再和 HTML / CSS 混在一起。

JavaScript 让网页“活过来”。它可以处理点击事件、切换样式、发起请求、读取返回数据，是前端里最直接和用户交互的一层。

  • 1没有 JS 的网页更像静态文档，有 JS 才能交互。
  • 2常见事件：click。
  • 3可以通过 `onclick` 改变元素样式。
  • 4也可以通过 `fetch` 发送 GET / POST 请求。
JavaScript 演示图
点击变色示例
html：
<h2 id="title">我是标题</h2>
<button onclick="changeColor()">点击变色</button>

JS：
//JS 部分：实现交互功能
//1. 切换标题颜色
function changeColor() {
    let title = document.getElementById("title");
    if (title.style.color === "red") {
        title.style.color = "blue";
    } else {
        title.style.color = "red";
    }
}

async function getRequest() {
  const url = "https://jsonplaceholder.typicode.com/users?id=1&name=test";
  const res = await fetch(url);
  const data = await res.json();
  return data;
}
05 / 后端基础

后端负责处理请求和业务判断

理解 GET 和 POST 的差别，是入门后端和接口交互最基础的一步。

后端接收前端传来的请求，处理逻辑后返回响应。不同请求方式适用于不同场景，查询用 GET，提交和修改通常用 POST。

  • 1GET：参数在 URL 上，适合查询、获取数据。
  • 2POST：参数放在 body 中，适合提交、新增、修改数据。
  • 3后端负责判断逻辑，不直接把数据写死在页面里。
  • 4接口的核心价值是把前端请求和后端处理连接起来。
对比
GET 与 POST
GET 适合拿数据，POST 适合传数据。安全性、缓存、长度限制、使用场景都不一样。
重点
请求 - 处理 - 响应
前端发送请求，后端判断并处理，最后返回结果给前端展示。
后端基础示意图
GET / POST 对比
| 对比维度 | GET 请求                      | POST 请求                  |
| -------- | ----------------------------- | -------------------------- |
| 传参方式 | 参数拼接在 URL 地址栏         | 参数放在请求体 body 中     |
| 数据长度 | 受浏览器URL长度限制，传参短小 | 无长度限制，可传大量数据   |
| 安全性   | 参数暴露在地址栏，不安全      | 参数隐藏在请求体，相对安全 |
| 缓存     | 默认会被浏览器缓存            | 默认不缓存                 |
| 使用场景 | 查询、获取数据（无数据修改）  | 提交、新增、修改数据       |
06 / 数据库基础

数据库负责长期保存数据

前端界面上的内容要真正落地，最终还是要靠数据库保存。

数据库是前端、后端之外的第三个关键部分。数据能永久保存，业务才能“下次打开还在”。SQLite、MySQL 都是常见的基础数据库。

  • 1SQLite：文件型数据库，不需要单独安装服务。
  • 2MySQL：常见的关系型数据库，适合项目中长期使用。
  • 3增删改查是数据库最基础、最必须掌握的能力。
  • 4没有数据库，很多系统只能算临时页面，无法持续保存信息。
SQL
四条必须掌握的语句
SELECT 查、INSERT 增、UPDATE 改、DELETE 删，是数据库入门的核心。
SELECT * FROM users; INSERT INTO users (name, age) VALUES ('张三', 25); UPDATE users SET name = '哈哈' WHERE name = '张三'; DELETE FROM users WHERE id = 1;
判断题
什么系统最需要数据库？
只要数据需要下次再打开还能保留，基本都离不开数据库。
